# Contributing to LMS Logger

Thank you for your interest in contributing.

## How to contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Write a focused test first for any new feature or bug fix.
4. Implement the code change to make the test pass.
5. Ensure the code is documented with docstrings and comments for non-obvious logic.
6. Run `pytest` and `mypy src/` before submitting a pull request.
7. Keep commits small and use clear, readable commit messages.

## Code style

- Follow PEP 8 formatting.
- Use type hints and static type checking.
- Prefer defensive coding and robust error handling.
- Keep the codebase maintainable and secure.

## Reporting issues

If you find a bug or want to suggest an improvement, open an issue with a clear description and reproduction steps.
